Detailed Description

Central aggregate object of the balancer features.

The Balancer_t aggregates:

  • hardware abstraction objects (by pointer, owned by main.c),
  • control objects (by value, owned by Balancer_t),
  • operating state (mode, flags, timers),
  • the editable parameters,
  • the public method table (function pointers).

Method dispatch follows the prototype-copy pattern: a const instance "Balancer" in balancer_t.c holds the function pointers, BalancerCreate() copies it into the user-supplied Balancer_t, then overwrites the HW pointers and calls init().

Lifetime: a single static Balancer_t named "bala" lives in main.c.
